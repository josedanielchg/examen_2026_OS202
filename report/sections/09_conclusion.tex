\section{Conclusion intermédiaire}

Les deux premières étapes installent désormais une base expérimentale cohérente pour la suite du projet :
\begin{itemize}
    \item une implémentation autonome de l'étape 1 dans \texttt{stages/01\_numba\_threads/}, dédiée au calcul partagé avec Numba ;
    \item une implémentation autonome de l'étape 2 dans \texttt{stages/02\_mpi\_display\_compute/}, dédiée à la séparation MPI entre affichage et calcul ;
    \item une stratégie de benchmark reproductible distinguant calcul local, coût bout-en-bout et surcoût de communication ;
\end{itemize}

L'étape 2 montre ensuite qu'il est possible de séparer proprement affichage et calcul avec MPI tout en gardant un surcoût de transfert très faible, de l'ordre de \SI{0.12}{ms} par pas seulement dans la configuration retenue.

L'intérêt principal de l'étape 2 est architectural : l'affichage ne vit plus dans le même processus que le calcul, et la communication MPI reste suffisamment légère pour préparer la prochaine étape, où plusieurs processus devront réellement se partager le calcul.
